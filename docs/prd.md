# PRD: SQL 자동 튜닝 AI 웹 서비스

## 1. 개요
DBA가 아닌 개발자가 SQL, 실행계획, 스키마 정보를 붙여넣어서 느린 원인을 파악하고 튜닝된 쿼리까지 받아볼 수 있는 웹 서비스

---

## 2. 타겟 유저
- SQL을 쓰지만 최적화는 잘 모르는 개발자
- 남이 짠 쿼리를 해석해야 하는 상황의 개발자
- DBA 없는 스타트업/사이드 프로젝트 개발자

---

## 3. 핵심 기능 (MVP)

### 입력 방식
- 쿼리 텍스트 붙여넣기
- 실행계획 텍스트 붙여넣기
- 스키마 정보 붙여넣기 (CREATE TABLE 문, 선택사항)
  - 쿼리 입력 시 사용된 테이블 자동 감지 → 해당 테이블 스키마만 입력 안내

### [분석]
- 쿼리 설명 (이 쿼리가 뭘 하는 쿼리인지)
- 실행계획 해석
- 느린 원인 추출 (구체적으로 → "조인 컬럼 타입 달라서 인덱스 못 탐" 수준)
- 디스크 I/O 표시

### [제안]
- 인덱스 제안 (어떤 컬럼에 왜 필요한지)
- 스키마 정보 있을 시 테이블 관계 기반 정밀 분석

### [생성]
- 튜닝된 쿼리 생성
- Original / Diff 형태로 보여주기

---

## 4. 분석 구조
룰 + AI 혼합
- **룰 기반** → 확정적인 문제 탐지 (타입 불일치, 풀스캔 등)
- **AI** → 설명 + 개선 제안 생성 (스키마 정보 있을 시 컨텍스트로 활용)

처리 순서:
1. 쿼리 파싱 + 사용 테이블 감지
2. 실행계획 분석
3. 룰 기반 문제 탐지
4. AI 개선 제안 (스키마 컨텍스트 포함)
5. 결과 반환

---

## 5. 지원 DB
- **MVP** → MSSQL
- **추후** → DB 직접 연결 (스키마 자동 수집), PostgreSQL, MySQL 추가

---

## 7. 추후 기능 (MVP 이후)
- 로그인 기능
- 분석 내역 저장
- 결제 / 플랜 관리 (무료 제한 + 유료 구독)

---

## 6. 차별화 포인트
- 실행계획 기반 분석 (단순 쿼리 리뷰 아님)
- "느립니다" 가 아닌 구체적 원인 제시
- 쿼리에서 사용된 테이블 자동 감지 → 필요한 스키마만 입력 안내
- 스키마 정보 기반 정밀 분석 (테이블 관계, 인덱스, 컬럼 타입 반영)
- 룰 + AI 혼합으로 정확도 확보
- Original / Diff 비교로 변경사항 직관적 확인
